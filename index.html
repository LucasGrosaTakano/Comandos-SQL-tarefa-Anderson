Guia Rápido de Comandos SQL

DDL (Data Definition Language)

A Linguagem de Definição de Dados é usada para definir e gerenciar a estrutura do banco de dados e seus objetos, como tabelas.

CREATE

Usado para criar objetos no banco de dados, como tabelas, views e índices.

CREATE TABLE Clientes (
    ID INT PRIMARY KEY,
    Nome VARCHAR(100),
    Email VARCHAR(100)
);

ALTER

Usado para modificar a estrutura de um objeto existente, como adicionar uma coluna a uma tabela.

ALTER TABLE Clientes
ADD COLUMN Telefone VARCHAR(20);

DROP

Usado para excluir permanentemente um objeto do banco de dados.

DROP TABLE Clientes;

DML (Data Manipulation Language)

A Linguagem de Manipulação de Dados é usada para gerenciar os dados dentro das tabelas.

INSERT

Adiciona novos registros (linhas) a uma tabela.

INSERT INTO Clientes (ID, Nome, Email)
VALUES (1, 'João Silva', 'joao.silva@email.com');

UPDATE

Modifica registros existentes em uma tabela.

UPDATE Clientes
SET Email = 'joao.novo@email.com'
WHERE ID = 1;

DELETE

Remove registros de uma tabela.

DELETE FROM Clientes
WHERE ID = 1;

Consultas e Cláusulas de Filtro

Consultas são usadas para recuperar dados das tabelas. Cláusulas de filtro permitem especificar com precisão quais dados você deseja ver, aplicando condições lógicas e busca por padrões.

SELECT

É o principal comando para consultar e recuperar dados de uma ou mais tabelas. Pode ser usado para selecionar colunas específicas ou todas as colunas.

-- Seleciona colunas específicas de uma tabela
        SELECT Nome, Email FROM Clientes;
        
        -- Seleciona todas as colunas da tabela
        SELECT * FROM Clientes;

WHERE

Filtra os registros de uma consulta, retornando apenas aqueles que satisfazem a uma condição específica. Pode ser usado com operadores de comparação como =, >, <, >=, <=, <> (diferente).

-- Seleciona clientes com ID maior que 10
        SELECT Nome, Email FROM Clientes WHERE ID > 10;

Operadores Lógicos (AND, OR, NOT)

São usados com a cláusula WHERE para combinar múltiplas condições.

    AND: Exibe um registro se todas as condições forem verdadeiras.
    OR: Exibe um registro se pelo menos uma das condições for verdadeira.
    NOT: Exibe um registro se a condição não for verdadeira.
-- Clientes de São Paulo que se chamam 'João'
        SELECT * FROM Clientes
        WHERE Cidade = 'São Paulo' AND Nome = 'João';
        
        -- Clientes que são de São Paulo OU do Rio de Janeiro
        SELECT * FROM Clientes
        WHERE Cidade = 'São Paulo' OR Cidade = 'Rio de Janeiro';

LIKE

É usado na cláusula WHERE para buscar por um padrão específico em uma coluna de texto. Utiliza caracteres curinga:

    % (Percentual): Representa zero, um ou múltiplos caracteres.
    _ (Underline): Representa um único caractere. 
-- Encontra qualquer cliente cujo nome começa com 'Ana'
        SELECT * FROM Clientes WHERE Nome LIKE 'Ana%';
        
        -- Encontra qualquer cliente cujo nome termina com 'Silva'
        SELECT * FROM Clientes WHERE Nome LIKE '%Silva';
        
        -- Encontra qualquer cliente cujo nome tenha 'Souza' em qualquer posição
        SELECT * FROM Clientes WHERE Nome LIKE '%Souza%';

Relacionamentos e JOINs

Relacionamentos conectam tabelas através de chaves. O comando JOIN é usado nas consultas para combinar linhas de duas ou more tabelas com base em uma coluna relacionada entre elas.

INNER JOIN

Retorna registros que têm valores correspondentes em ambas as tabelas. É o tipo de junção mais comum.

-- Supondo uma tabela Clientes e uma tabela Pedidos
        -- Retorna o nome do cliente e o ID do pedido
        SELECT Clientes.Nome, Pedidos.ID
        FROM Clientes
        INNER JOIN Pedidos ON Clientes.ID = Pedidos.ClienteID;

LEFT JOIN (ou LEFT OUTER JOIN)

Retorna todos os registros da tabela da esquerda (a primeira mencionada) e os registros correspondentes da tabela da direita. Se não houver correspondência, o resultado é NULL no lado direito.

-- Retorna todos os clientes e seus pedidos, se tiverem
        SELECT Clientes.Nome, Pedidos.ID
        FROM Clientes
        LEFT JOIN Pedidos ON Clientes.ID = Pedidos.ClienteID;

RIGHT JOIN (ou RIGHT OUTER JOIN)

Retorna todos os registros da tabela da direita (a segunda mencionada) e os registros correspondentes da tabela da esquerda. Se não houver correspondência, o resultado é NULL no lado esquerdo.

-- Retorna todos os pedidos e o nome do cliente correspondente
        SELECT Clientes.Nome, Pedidos.ID
        FROM Clientes
        RIGHT JOIN Pedidos ON Clientes.ID = Pedidos.ClienteID;

Tipos de Relacionamento

Um-para-Muitos (1:N): O mais comum. Um cliente pode ter muitos pedidos.
Um-para-Um (1:1): Um usuário e seu perfil de configurações.
Muitos-para-Muitos (N:N): Requer uma tabela associativa. Alunos e Cursos.
